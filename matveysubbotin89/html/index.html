Welcome to user interface for "bankomat" program

Description of the program

Did you think about ATM working.

  • How is he calculate number of banknotes according to your request?
  • What will he do when one of banknotes basket is empty?
  • What is happens when you type number, which is higher as available?
  • What is happened if you enter zero?

This program will answer on this and more other questions

Input data

How many money user needs. This information are transferring to program as input data.
User should type number in field. The number must be integer and uses available nominals banknotes as dividers.

Output data

Program will return dictionary, dictionary includes every nominals of banknotes as key and number of banknotes as value of this key.
The result will have been showed like this {5: 1, 10: 6, 20: 10, 50: 10, 100: 1, 200: 0, 500: 1, 1000: 1}. It does mean user entered nuber 2365.
If you summ all banknotes (key*value), the result will eqvel to input number.

Feature of the program

As you can see, the returned value is not normal. Programs is trying to give money with minimal nominal of banknotes at first, and rest will be calculated from highest to lowest nominals of banknotes.

Limitations of program

Program does not work with banknotes lowest as 5 grivnas. If user enter number which is could to divide on smaller number as five.
Program will return error message : "Sorry, value is not correct."

Engineer can change quantity of smallest banknotes, the default value is 10(ten) for banknotes: 5, 10, 20, 50.
